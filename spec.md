# 開発者用仕様書（仮）

## 1. 全体方針
- ベースとなるシステム：京葉線システム (`keiyo2`)
- データ管理：メモリ上の変数（配列）にて管理する
- 構造：Node.js + Express + EJS
- 共通仕様：
    - 各リソースに対し、REST API風のURL設計を行う
    - 一覧、詳細、追加、編集、削除の機能を実装する

---

## 2. システム1：授業課題管理システム

### 概要
大学の授業課題を登録し、提出期限や所要時間を管理するシステム。

### データ構造
| 論理名 | 変数名(案) | 型 | 説明 |
| :--- | :--- | :--- | :--- |
| ID | `id` | int | 自動連番 |
| 科目名 | `code` | string | 授業の科目名（例：Webプログラミング） |
| 課題内容 | `name` | string | 課題のタイトル（例：第10回レポート） |
| 提出期限 | `passengers` | int | 提出期限（例：20251210） |
| 所要時間 | `distance` | int | 予想される所要時間（分） |

### リソースとページ遷移
- ベースURL: `/tasks`

| 機能 | HTTPメソッド | URL | 処理内容 | 遷移先/表示 |
| :--- | :--- | :--- | :--- | :--- |
| 一覧表示 | GET | `/tasks` | 登録済み課題の一覧を表示 | `tasks_list.ejs` |
| 詳細表示 | GET | `/tasks/:id` | 特定の課題の詳細情報を表示 | `tasks_detail.ejs` |
| 追加画面 | GET | `/tasks/create` | 新規登録フォームを表示 | `tasks_create.html` |
| 追加処理 | POST | `/tasks/create` | フォームデータを配列に追加 | `/tasks` へリダイレクト |
| 編集画面 | GET | `/tasks/edit/:id` | 既存データの編集フォームを表示 | `tasks_edit.ejs` |
| 更新処理 | POST | `/tasks/update/:id` | 指定IDのデータを更新 | `/tasks` へリダイレクト |
| 削除処理 | GET | `/tasks/delete/:id` | 指定IDのデータを削除 | `/tasks` へリダイレクト |

## 3. システム2：参考文献管理システム

### 概要
レポートや論文作成に使用する参考文献を記録・管理するシステム。

### データ構造
| 論理名 | 変数名(案) | 型 | 説明 |
| :--- | :--- | :--- | :--- |
| ID | `id` | int | 自動連番 |
| 著者名 | `code` | string | 著者の名前 |
| 書籍名 | `name` | string | 本のタイトル |
| 発行年 | `passengers` | int | 出版された年（例：2024） |
| 重要度 | `distance` | int | 5段階評価など |

### リソースとページ遷移
- ベースURL: `/references`
- ページ遷移・機能はシステム1と同様（`/tasks` を `/references` に読み替える）

## 4. システム3：家計簿システム

### 概要
日々の支出を記録し、金額や用途を管理するシステム。

### データ構造
| 論理名 | 変数名(案) | 型 | 説明 |
| :--- | :--- | :--- | :--- |
| ID | `id` | int | 自動連番 |
| カテゴリ | `code` | string | 食費、交通費、趣味など |
| 商品名 | `name` | string | 購入したもの（例：お弁当） |
| 金額 | `passengers` | int | 支払い金額（円） |
| 日付 | `distance` | int | 購入日（例：1205） |

### リソースとページ遷移
- ベースURL: `/household`
- ページ遷移・機能はシステム1と同様（`/tasks` を `/household` に読み替える）
